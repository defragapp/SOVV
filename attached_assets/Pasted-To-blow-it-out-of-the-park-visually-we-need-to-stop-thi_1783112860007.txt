# Phase 23: Cinematic OS Overhaul (Visual Apex)

## Core Objective
The platform currently looks like a series of web pages. We are evolving it into a "Cinematic OS." Every visual asset, transition, and element must feel like physical hardware.

## Strict System Constraints
1. **Zero-Edge Architecture:** Absolutely no containers or cards. Content must float on the ambient environment.
2. **Motion-Induced Depth:** All transitions must use Z-axis depth (scaling). No more flat slides or fades.
3. **Hardware Materiality:** Everything must be dark, glass, or amber light. No "web" colors. 

---

## Execution Steps

### 1. The Global "Glass-Bleed" Effect
- Replace all standard border/shadow styles with a **Global Glass-Bleed**. 
- Any element that requires a boundary must use a subtle, 1px-wide amber light-leak (`border border-amber-500/10`) instead of a generic white/gray border. 
- This simulates the effect of light bleeding through the edges of a glass slab.

### 2. Micro-Cinematics (The "OS" Feel)
- **Typography:** Fraunces must be used for all structural headers. Geist for everything else. Increase letter-spacing on all `JetBrains Mono` tags by `0.05em` to make them feel like terminal readouts.
- **Micro-Animations:** Every time a user types, the background volumetric light should react (a 0.5% scale pulse). This makes the *entire system* feel aware of the user's input.
- **The "Focus" State:** When a user enters a space (Defrag/Covenant/Alignment), the ambient background should subtly shift color temperature (slightly warmer in Covenant, slightly cooler in Alignment) to signal a change in "environment."

### 3. Final Polish: The "Physicality" Audit
- **Remove all "web" UI:** If a component looks like it belongs on a marketing site, delete it or rewrite it.
- **Action Buttons:** Ensure every single action button uses a `1px` inner-glow effect and `active:scale-[0.97]` physics.
- **Negative Space:** Double the current padding across the entire platform. OS-level interfaces breathe more than web apps. Give the typography room to exist on the canvas.

Execute this "Cinematic OS" overhaul now. Focus on the physical presence of the components. I want this to look like an operating system you would find on a custom device.
